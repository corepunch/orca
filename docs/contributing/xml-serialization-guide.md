# XML Serialization — Implementation Guide

This document explains how to implement or extend XML serialization of the object graph.
It was written after actually implementing `FS_SerializeObjectToXmlString` in
`source/filesystem/fs_xml_write.c`, so every problem here is real, not theoretical.

---

## Quick orientation

The deserialization path lives in `source/filesystem/fs_xml.c` and
`source/filesystem/fs_xml_inline.c`. The serializer mirrors it in
`source/filesystem/fs_xml_write.c`. The public entry point is:

```c
LPSTR FS_SerializeObjectToXmlString(struct Object const *object);
// Returns heap-allocated XML; caller must free() it.
// Declared in include/orca.h.
```

---

## Codebase layout you must understand first

### Object and component model

An `Object` is a bag of *components* (linked list of `struct component`).
Each component corresponds to a class (e.g. `Node`, `Node2D`, `TextBlock`).
Multiple-inheritance objects have one component per ancestor, added by
`OBJ_AddComponent` which recurses into parent classes after adding itself.

```
Object
 └─ components (linked list)
     ├─ TextBlock component   ← most-derived, first in list → OBJ_GetClassName()
     ├─ TextBlockConcept component
     └─ Node2D component
         └─ Node component
```

**The first component's class name is the XML tag.** `OBJ_GetClassName()` returns
`cmp->pcls->ClassName` for the first component, which is correct.

### PropertyType descriptor

Every component class has an array of `PropertyType` descriptors
(`ClassDesc.Properties[0..NumProperties-1]`).  The layout, as documented in
`docs/architecture/object-component-system.md`:

```c
struct PropertyType {
  const char  *Name;            // bare name, e.g. "Width"
  const char  *Category;        // component name, e.g. "Node2D"
  enum DataType DataType;        // kDataTypeFloat, kDataTypeString, …
  const char  *TypeString;       // struct/class name for struct/object props
  const char **EnumValues;       // null-terminated array for enum props
  uint32_t     ShortIdentifier;  // FNV1a("Width")
  uint32_t     FullIdentifier;   // FNV1a("Node2D.Width")
  uint32_t     Offset;           // offsetof inside component user-data block
  uint32_t     DataSize;         // sizeof the field (or sizeof element for arrays)
  bool_t       IsArray;
  bool_t       IsInherited;
  // … AffectLayout, AffectRender, IsReadOnly (not always present in codegen output)
};
```

These are generated by the `DECL` / `ARRAY_DECL` macros in `include/codegen.h`
and emitted into `*_export.c` files by the codegen tool.

## Walking all properties of an object

Use `OBJ_EnumClassProperties` (declared in `include/orca.h`):

```c
void OBJ_EnumClassProperties(
    struct Object *object,
    void (*fnProc)(struct Object const *,
                   struct PropertyType const *,
                   struct ClassDesc const *,
                   void const *component_data,
                   void *param),
    void *param);
```

This walks every component and every property within it.  Pass a context struct
via `param`; **do not use a block literal** — the function takes a plain C
function pointer, not a clang block.

## Converting a property value to string

There is **no public** `format_property` / `PropertyToString` function.
`PDESC_Print` exists in `property_core.c` but is (a) not declared in any
header, and (b) broken for `kDataTypeStruct` (treats struct bytes as a raw
float array, ignoring field layout).

You must implement the conversion yourself.  The type switch is:

| DataType | How to format |
|---|---|
| `kDataTypeBool` | `"true"` / `"false"` via `*(int*)value` |
| `kDataTypeInt` | `snprintf("%d", *(int*)value)` |
| `kDataTypeEnum` | `pd->EnumValues[*(int*)value]` |
| `kDataTypeFloat` | `snprintf("%g", *(float*)value)` |
| `kDataTypeColor` | `#RRGGBB` or `#AARRGGBB`; value is `float[4]` RGBA in 0–1 range |
| `kDataTypeString` | `*(char**)value` (already a heap string) |
| `kDataTypeStruct` | Recurse: call `OBJ_FindStructDesc(pd->TypeString)`, iterate `.Properties`, format each field, space-separate positionally |
| `kDataTypeObject` | Needs special handling (see below) |
| `kDataTypeEvent` | Not serializable as an attribute |

### Struct properties

`OBJ_FindStructDesc(pd->TypeString)` returns a `StructDesc` with its own
`Properties` array.  Each field is itself a `PropertyType` with an `Offset`
relative to the struct base.  Serialize fields space-separated in declaration
order to match the positional inline syntax the deserializer accepts:

```c
// Serializes: "10 20" (e.g. a Vec2 with x=10, y=20)
FOR_LOOP(i, sd->NumProperties) {
    struct PropertyType const *field = &sd->Properties[i];
    format_value(field, (char*)value + field->Offset, &out);
}
```

### Object-reference properties (`kDataTypeObject`)

Cannot be emitted as a flat attribute.  Instead, emit a child XML element
named after the property, containing the referenced object's serialized form:

```xml
<Node2D>
  <Node.DataContext>          <!-- property element -->
    <DataObject Name="..."/>  <!-- the referenced object -->
  </Node.DataContext>
</Node2D>
```

Use `Category.Name` as the element name when the bare `Name` collides with
a registered class name (`OBJ_FindClass(pd->Name) != NULL`).

### Array properties (`pd->IsArray == TRUE`)

Emit a property-element wrapper, then one child per item:

```xml
<Node>
  <Node.Triggers>
    <EventTrigger .../>
    <EventTrigger .../>
  </Node.Triggers>
</Node>
```

The array value layout is `{ void *items; int count; }` packed contiguously.
Read it as:

```c
void *items = *(void **)raw_value;
int count   = ((int *)raw_value)[sizeof(void*) / sizeof(int)];
```

## Binding expressions

**The original expression string is not stored after compilation.**
`PROP_SetBinding` calls `Token_Create(expression)` which builds a token tree
in memory and discards the source text.  The `struct Binding` (defined only
in the private `property_internal.h`) holds only:

```c
struct Binding {
  struct token  *token;       // compiled expression
  struct Property *property;  // back-pointer
  uint32_t updateFrame;
};
```

To serialize bindings round-trip you must store the expression string at parse
time.  The cleanest fix is to add an `expression` field to `struct Binding` and
populate it in `PROP_SetBinding`:

```c
// in property_internal.h
struct Binding {
  struct token    *token;
  struct Property *property;
  LPSTR            expression;   // ADD THIS
  uint32_t         updateFrame;
};

// in PROP_SetBinding:
p->binding->expression = expression ? strdup(expression) : NULL;
```

Until that change lands, the serializer emits a `<BindingExpression
Target="Category.Name"/>` stub with no expression text — the property name is
preserved but the expression is lost.

**Detection:** `PROP_HasProgram(prop)` returns `TRUE` if a binding is
compiled.  Skip such properties from attribute serialization and instead emit
the binding stub.

## StyleClass / CSS class attribute

The `class` attribute (maps to `StyleClass` / CSS selector) is **not stored as
a normal property**.  It is processed by the `StyleController` component and
stored internally as a linked list of `struct style_class_selector` (declared
in `core_local.h`, a private header).

`OBJ_EnumStyleClasses` (also in `core_local.h`) is not in the public API and
yields parsed selector structs including pseudo-state flags and opacity
suffixes, not the original raw string.

**To make StyleClass serializable, add a public accessor:**

```c
// Option A: raw string accessor (simplest)
ORCA_API lpcString_t OBJ_GetRawStyleClasses(struct Object *object);

// Option B: store the raw string as a regular string property on Node
// and update it whenever StyleController processes an AddClasses message
```

Until either option is implemented, the `class` attribute is silently dropped
during serialization.

## Text content

Objects can have text content set via `OBJ_SetTextContent`.  Serialize it as
the element's text node.  When text content is present, child objects are not
expected (the deserializer returns early after setting text content):

```c
lpcString_t text = OBJ_GetTextContent(object);
if (text && *text) {
    xmlNodeSetContent(node, XMLSTR(text));
    return; // no children
}
```

## Prefab / template objects (`OF_TEMPLATE` flag)

Objects loaded via `LayerPrefabPlaceholder` / `ObjectPrefabPlaceholder` are
re-loaded from their source file at runtime.  Their `SourceFile` field
(`OBJ_GetSourceFile`) holds the original path.

The correct round-trip serialization emits the placeholder element, not the
expanded content:

```xml
<LayerPrefabPlaceholder PlaceholderTemplate="Example/Prefabs/Card" .../>
```

**Detection:** `OBJ_GetFlags(object) & OF_TEMPLATE` and `OBJ_GetSourceFile`.

The current implementation does not handle this — it serializes the expanded
object tree instead.  This is lossy.

`FS_SerializeObjectToXmlString` is declared in `include/orca.h`.  It was not
originally declared there (only a stub in `fs_xml.c`) — this was corrected
when implementing the serializer.

## Known limitations (summary)

| Issue | Status |
|---|---|
| `PDESC_Print` broken for structs, not in header | **Fixed** — `OBJ_FormatPropertyValue` in `include/orca.h` |
| Binding expression text not stored post-compile | **Fixed** — `PROP_GetBindingExpression` / `struct Binding.Expression` |
| StyleClass not in public API | **Fixed** — `OBJ_GetRawStyleClasses` in `include/orca.h` |
| Prefab objects serialized as expanded tree | **Fixed** — `OF_TEMPLATE` detection emits `<LayerPrefabPlaceholder>` |
| No `IsReadOnly` flag respected | **Fixed** — `pd->IsReadOnly` skip in `serialize_property_cb` |
| Block literal vs. function pointer | **Fixed** — plain C callback with `struct serialize_ctx *param` |
| `LPSTR` return: `strdup`'d, free with `free()` | Correct |

## Minimal working example

```c
// Serialize an object to an XML string
struct Object *screen = FS_LoadObjectFromXml("Screens/Main");
LPSTR xml = FS_SerializeObjectToXmlString(screen);
if (xml) {
    printf("%s\n", xml);
    free(xml);
}

// Round-trip (loses bindings and CSS class)
LPSTR xml2 = FS_SerializeObjectToXmlString(screen);
struct Object *copy = FS_LoadObjectFromXmlString(xml2);
free(xml2);
```
